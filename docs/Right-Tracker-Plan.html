📄 What is a Ticket Management System?

A centralized platform where employees raise IT issues, IT staff resolves them, and management tracks everything.

Current Problems

Lost Requests

IT requests via email, phone, walk-ins get forgotten or misplaced

No Visibility

Employees have no way to track the status of their requests

No Accountability

No record of who handled what or how long it took

No Performance Data

Management cannot measure IT team efficiency or identify recurring issues

Our Solution: Right Tracker

Centralized Ticketing

Every request logged in one place — nothing gets lost

Real-time Tracking

Employees track their tickets like tracking a delivery order

Full Accountability

Every action is tracked — who did what, when

Reports & Analytics

Dashboards and reports for data-driven decisions

👥 User Roles

Three distinct roles with different access levels and responsibilities.

Admin

  • Full system access
  • Manage users and roles
  • Assign tickets to IT staff
  • View all tickets and reports
  • Close resolved tickets
  • System configuration

IT Staff

  • View assigned + open tickets
  • Self-assign unassigned tickets
  • Update ticket status
  • Add internal notes
  • Reply to employees
  • View own performance

Employee

  • Submit new tickets
  • Track own tickets only
  • Add comments / replies
  • Upload attachments
  • Reopen resolved tickets

🔒 Security Features

Enterprise-grade security based on proven patterns from WisRight projects.

Authentication

JWT tokens (24h expiry), bcrypt password hashing (10 rounds), account lockout, no user enumeration

API Protection

Helmet.js security headers, CORS restriction, rate limiting (100 req/15min), request size limits

Data Protection

No plain-text passwords, SQL injection prevention (ORM), input validation, environment-based secrets

Infrastructure

HTTPS via Coolify/Traefik, Docker isolation, Content Security Policy, secure error handling
